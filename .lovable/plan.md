## 목표
5개 메뉴를 **주문 목록(작업지시번호 클릭) → 주문 상세** 2단계 구조로 통일.

작업지시번호 = `orders.external_order_id`. 클릭 시 `?orderId=...` 쿼리로 상세 진입, 좌상단 "← 목록" 버튼으로 복귀.

## 공통 패턴
- 새 컴포넌트 `src/components/OrderListPicker.tsx` — 주문 목록 테이블 (작업지시번호 / 트윈커 / 수량 / 진행상황 / 납기일)
- 각 페이지: URL `?orderId` 없으면 목록, 있으면 상세
- 상세는 해당 `orderId`로 필터링된 기존 UI 재사용

## 페이지별 변경

### 1. 카드 QR코드 수동검사 (`CardQrInspection.tsx`)
이미 목록>상세 구조 존재 → 작업지시번호 컬럼을 클릭 가능한 버튼으로, 상세는 그대로 유지

### 2. 카드 사진 검사 (`CardPhotoInspection.tsx`)
이미 목록>상세 구조 존재 → 동일하게 클릭 인터랙션 통일

### 3. 티셔츠 부착 작업 (`TshirtWork.tsx`)
현재 단일 화면. → 상단에 주문 목록 추가, 작업지시번호 클릭 시 해당 주문 항목만 필터링된 스캔 작업 화면 표시

### 4. 생산/포장 모니터링 (`ProductionMonitor.tsx`)
파이프라인 탭에 주문 목록 추가. 작업지시번호 클릭 시 해당 주문의 7단계 진행 상세(production_tracking + shipments) 표시. 기계/CCTV 탭은 그대로.

### 5. 배송 관리 (`Shipping.tsx`)
큐 형태 → 주문 목록(스캔 진행률 컬럼) 표시. 작업지시번호 클릭 시 기존 `/shipping/scan/:orderId` 페이지로 이동(또는 인라인 표시).

## 기술 노트
- 데이터는 `useOrders()` 재사용
- `useSearchParams`로 orderId 관리 → 뒤로가기 자연스럽게 동작
- 진행률 컬럼은 각 페이지 도메인에 맞는 카운트 (스캔 완료/총 / 부착 완료/총 등)
- 디자인 토큰 유지, i18n(KO/ZH) 유지
